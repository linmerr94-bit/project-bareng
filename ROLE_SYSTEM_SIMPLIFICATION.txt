## SIMPLIFIED ROLE SYSTEM (3 ROLES ONLY)

The system now uses ONLY 3 roles:

1. **CUSTOMER** (role='customer')
   - Can browse products
   - Can add items to cart
   - Can checkout and make orders
   - Can view order history and profile
   
2. **PENJUAL / SELLER** (role='brand')
   - Can add/edit/delete products
   - Can create multiple products without approval
   - Can manage their own brand/store
   - Can view their products and orders
   - Auto-creates Brand profile if not exists
   
3. **ADMIN** (role='admin')
   - Full platform management
   - Can view all users, products, orders
   - Can approve/reject content if needed

## CHANGES MADE

### 1. VENDOR_DASHBOARD_VIEW - SIMPLIFIED ROLE CHECK ✅

**File:** master_products/views.py (line ~844)

**BEFORE:**
```python
# Check VendorRequest status='Approved' - STRICT
vendor_request = VendorRequest.objects.filter(
    user=request.user,
    status='Approved'
).first()

if not vendor_request:
    messages.error(request, 'Akses ditolak! Anda harus menjadi vendor yang disetujui...')
    return redirect('master_products:vendor_dashboard')

brand, created = Brand.objects.get_or_create(
    user_id=request.user,
    defaults={
        'brand_name': vendor_request.vendor_name,  # Depends on VendorRequest
        'description': vendor_request.description,
        'status': 'approved'
    }
)
```

**AFTER:**
```python
# Check role='brand' - SIMPLE
if request.user.role != 'brand':
    messages.error(request, 'Akses ditolak! Hanya penjual yang dapat mengakses dashboard ini.')
    return redirect('master_products:login')

# Auto-create brand if not exists
brand, created = Brand.objects.get_or_create(
    user_id=request.user,
    defaults={
        'brand_name': request.user.full_name or request.user.username.upper(),
        'description': f'Toko {request.user.username}',
        'status': 'approved'  # Always approved - no waiting!
    }
)
```

**Impact:**
- ✅ Removed VendorRequest dependency
- ✅ Sellers can immediately access dashboard
- ✅ No approval waiting needed
- ✅ Brand auto-created with default name
